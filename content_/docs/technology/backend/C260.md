# 소개

`OpenSearch` 인덱스를 생성하는 세팅 방법을 작성한 문서입니다. (프로젝트 초기 설정 시)

`CloudFormation` 을 통해 AWS 리소스가 생성된 상태여야 합니다. /w `serverless.yml`

---

## 1. Search-API 살펴보기

`search-api.ts` 를 확인합니다.

`SearchAPIController`의 API를 호출하여 해당 프로젝트의 Elastic Search 인덱스 확인 및 생성을 할 수 있습니다.



각 API endpoint는 다음 기능을 제공합니다.





💡 **init-elastic** : 인덱스 생성

**destroy-elastic** : 인덱스 삭제

**desc-elastic** : 인덱스 조회 (상세정보)



- OpenSearch sse 터널링을 연결해 준 뒤,init-elasticAPI를 호출하여 인덱스를 생성합니다.
```javascript
# 인덱스 생성하기
$ http ':8888/search/init-elastic/hello'

# 인덱스 조회하기
$ http ':8888/search/desc-elastic/hello'

```text
---

## 2. Mappings 확인하기

- 인덱스에 저장될 문서의 필드와 해당 필드의 데이터 타입의 정의를 확인해야 합니다.
- 데이터를 생성하고, 리스트 조회가 잘 되는지 확인합니다.
- mappings를 확인하여 타입이 잘 매핑되어 있는지 확인합니다.
```javascript
# 데이터 생성 테스트
$ echo { "name": "lemon", "age": 10 } http POST ':8888/tests/0'

# 목록 조회 테스트 (동기화가 잘 들어와있는지 확인)
$ http ':8888/tests'

# 인덱스 조회 후 mappings > properties 확인하기
$ http ':8888/search/desc-elastic/hello'
{
  ...,
  mappings: {
    ...,
    properties: {
      ...
    }
  }
}

```text

## 3. 부록

- Error Handling
  - 400 MAPPER PARSING - mapper_parsing_exception특정 속성에 대하여 초기에 매핑된 타입과 다른 타입이 들어왔을 경우 발생하는 에러입니다.
매핑이 잘못된 부분을 찾아, 코드에서 타입 변환에 관한 로직을 수정해야 합니다.
re-index를 하는 방법도 있지만, 사소한 타입 문제라면 이 방식을 추천합니다.
  - 400 UNKNOWN - connect ECONNREFUSED ::1:PORT_NUMssh 터널링을 하지 않았을 때 발생하는 에러입니다. 접속 해준 뒤 재시도 하세요.
